1. 트랜잭션 기본 개념 이해
| 용어                     | 설명                                | 예시                                     |
| ---------------------- | --------------------------------- | -------------------------------------- |
| **트랜잭션 (Transaction)** | 데이터베이스에 대한 작업을 하나의 “논리적 단위”로 묶은 것 | “주문 등록 → 결제 차감 → 재고 감소”를 하나의 트랜잭션으로 처리 |
| **Commit**             | 트랜잭션을 확정 → DB에 영구 반영              | 주문 등록 성공                               |
| **Rollback**           | 트랜잭션을 취소 → DB를 원래 상태로 복구          | 결제 실패 시 모든 변경 취소                       |

2. 트랜잭션의 4대 원칙
| 원칙                        | 의미                           | 예시                                |
| ------------------------- | ---------------------------- | --------------------------------- |
| **A - Atomicity (원자성)**   | 전부 수행되거나 전혀 수행되지 않음          | 주문 도중 오류 발생 시 전체 롤백               |
| **C - Consistency (일관성)** | 트랜잭션 전후로 DB 규칙이 유지           | 재고가 음수가 되지 않음                     |
| **I - Isolation (고립성)**   | 동시에 여러 트랜잭션이 실행돼도 서로 간섭하지 않음 | 다른 사용자의 주문이 아직 커밋되지 않은 데이터에 접근 불가 |
| **D - Durability (지속성)**  | 커밋된 내용은 영구적으로 저장됨            | 서버 재시작 후에도 데이터 유지                 |

3. 트랜잭션 전파(Propagation)
| 전파속성               | 설명                         | 예시                |
| ------------------ | -------------------------- | ----------------- |
| **REQUIRED (기본값)** | 이미 트랜잭션이 있으면 합류, 없으면 새로 생성 | ✅ 대부분의 경우 사용      |
| **REQUIRES_NEW**   | 기존 트랜잭션을 중단하고 새 트랜잭션 시작    | 로그 기록 등 별도로 처리할 때 |
| **MANDATORY**      | 반드시 기존 트랜잭션이 있어야 함         |                   |
| **NESTED**         | 내부 트랜잭션 (부분 롤백 가능)         |                   |

4. 트랜잭션 격리수준(Isolation Level)
| 수준                   | 특징                         | 문제 방지                  |
| -------------------- | -------------------------- | ---------------------- |
| **READ_UNCOMMITTED** | 아직 커밋 안 된 데이터 읽기 허용        | 거의 안 씀                 |
| **READ_COMMITTED**   | 커밋된 데이터만 읽음                | Dirty Read 방지          |
| **REPEATABLE_READ**  | 같은 트랜잭션 내에서는 같은 행 읽기 결과 보장 | Non-repeatable Read 방지 |
| **SERIALIZABLE**     | 완전 순차 실행                   | 가장 안전하지만 느림            |
※Spring에서는 @Transactional(isolation = Isolation.READ_COMMITTED) 등으로 설정 가능.
